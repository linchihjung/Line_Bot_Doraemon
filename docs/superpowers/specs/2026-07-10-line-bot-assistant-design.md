# LINE 個人助理 Bot 設計規格

日期：2026-07-10
狀態：待使用者審閱

## 目標

建立一個以 LINE 為入口的個人助理 Bot，提供三類核心能力：

1. 像 ChatGPT 一樣的隨問隨答與多輪追問。
2. 個人待辦與一次性提醒。
3. 使用者明確同意後才建立的個人長期記憶。

第一版支援使用者本人與少數信任的家人或朋友。每位使用者的資料必須完全分離。

## 已確認的產品決策

- 可接受免費額度 API，但不希望有固定月費或信用卡壓力。
- 第一版包含一般問答、待辦、個人記憶與提醒。
- 使用者不要求 Mac mini 常駐服務；Mac mini 通常會睡眠。
- 家中網路沒有固定 IP。
- 可接受將一般內容傳送至第三方免費 LLM，但需要敏感資訊警告與避免儲存設計。
- 技術選擇以免費部署、低維護成本為優先。
- 一般聊天不自動建立長期記憶。
- 明確指令一定執行；自然對話只有在意圖非常明確時才建議儲存。
- 提醒第一版到期只推播一次，之後標記為已提醒。
- 每位使用者可設定自己的時區，預設為 `Asia/Taipei`。
- 一般問答只保留最近幾輪對話，不長期保存完整聊天紀錄。

## 架構

第一版採雲端優先架構：

```text
LINE Messaging API
        |
        v
Cloudflare Worker
   |       |       |
   |       |       +--> Gemini API：問答與意圖判斷
   |       +----------> D1：使用者、待辦、記憶、提醒、短期對話
   +------------------> Cron：掃描到期提醒並推播
```

Cloudflare Worker 提供固定 HTTPS webhook，不依賴家中固定 IP，也不依賴 Mac mini 保持喚醒。Mac mini 僅作為開發環境，未來可選擇透過 Cloudflare Tunnel 連接本機 Ollama，但不納入第一版的必要路徑。

LINE 的訊息接收流程是由 LINE Platform 將 webhook 傳給 Bot server，再由 Bot server 回覆或推播訊息。

## 訊息處理

Worker 收到訊息後依序處理：

1. 驗證 LINE webhook 簽章。
2. 以 LINE user ID 識別使用者。
3. 檢查敏感資訊。
4. 先處理明確指令與可安全解析的提醒格式。
5. 其他訊息交給 Gemini 判斷意圖或產生一般問答回覆。
6. 驗證模型輸出的結構化結果。
7. 只有通過驗證後才寫入 D1。
8. 回覆 LINE 使用者。

Gemini 不直接操作資料庫，只能回傳允許的意圖與欄位，例如：

```json
{
  "intent": "create_reminder",
  "content": "繳電費",
  "due_at": "2026-07-11T09:00:00+08:00"
}
```

## 使用方式

### 一般問答

支援例如：

- 「幫我解釋量子糾纏」
- 「幫我寫一封請假信」
- 「改成比較正式一點」

一般問答會使用最近幾輪對話作為上下文，因此可以自然追問，但不會自動寫入長期記憶。

### 待辦

支援例如：

- 「我今天要做報告」
- 「列出我的待辦」
- 「完成做報告」

### 記憶

支援例如：

- 「記住我喜歡喝無糖茶」
- 「列出我的記憶」
- 「忘記我喜歡喝無糖茶」
- 「刪除所有資料」

如果一般對話內容看起來可能是長期偏好，Bot 只詢問是否要記住，得到同意後才儲存。

### 提醒

支援例如：

- 「提醒我明天早上 9 點繳電費」
- 「我的提醒」
- 「取消尚未推播的繳費提醒」

建立提醒後，Bot 必須回覆確認內容、日期、時間與時區。第一版提醒到期後推播一次並標記為已推播；重複提醒、完成確認與延後提醒留待後續版本。

## 資料模型

### `users`

保存 LINE user ID、顯示名稱、時區、建立時間與更新時間。

### `todos`

保存使用者 ID、待辦內容、狀態、建立時間、完成時間。

### `reminders`

保存使用者 ID、提醒內容、原始時區、標準化時間、狀態、推播時間與建立時間。

### `memories`

保存使用者 ID、記憶內容、來源、建立時間與更新時間。只接受使用者明確要求或確認的內容。

### `conversation_messages`

保存使用者 ID、最近幾輪訊息、角色、內容與建立時間。應設定保留上限或定期清理，避免變成完整聊天歷史資料庫。

所有使用者資料表都必須以 `user_id` 作為查詢條件；第一版不提供跨使用者資料查詢或共享。

## 隱私與安全

- 驗證 LINE webhook 簽章，拒絕未授權請求。
- 使用 Cloudflare secrets 保存 LINE token、LINE secret、Gemini API key。
- 偵測密碼、信用卡號、身分證字號等敏感資訊。
- 敏感資訊不寫入長期記憶，也不放入錯誤紀錄。
- 對使用者說明免費 Gemini 方案可能將內容用於產品改善。
- 一般問答與長期記憶分離，使用者可分別清除。
- D1 查詢必須使用目前使用者的 ID，避免越權讀取。
- 錯誤記錄只保存必要的錯誤類型與事件 ID，不保存完整訊息內容。

## 提醒排程

Cloudflare Cron 定期掃描尚未推播且已到期的提醒：

1. 以 UTC 儲存標準化時間。
2. 根據提醒的時區在建立時完成轉換。
3. 找出到期且未推播的提醒。
4. 透過 LINE Push Message API 推播。
5. 推播成功後標記為已推播。
6. 推播失敗時保留未推播狀態，並記錄可重試的錯誤。

為避免 Cron 重複執行造成重複推播，更新狀態時必須具備可辨識的條件，並使用事件或狀態欄位做冪等處理。

## 失敗行為

- Gemini 暫時無法使用：回覆目前無法回答，請稍後再試。
- 日期或時間不完整：要求補充必要資訊，不建立提醒。
- 模型輸出不符合 schema：視為無法判斷，不寫入資料庫。
- LINE webhook 重複送達：使用事件 ID 避免重複處理。
- D1 寫入失敗：不回覆成功，告知使用者操作未完成。
- 推播失敗：提醒保留未推播狀態，供下一次排程重試。

## 第一版驗收條件

- LINE webhook 能正常接收並回覆訊息。
- 一般問答可進行最近幾輪的多輪追問。
- 待辦、提醒、長期記憶與一般問答能正確分流。
- 明確記憶指令能儲存、查詢與刪除。
- 不同使用者無法讀取彼此的待辦、提醒、記憶或短期對話。
- 提醒會依使用者時區計算，到期只推播一次。
- Mac mini 睡眠時，雲端 webhook 與提醒仍可運作。
- 敏感資訊不會建立長期記憶，也不會進入錯誤紀錄。
- Gemini 或 D1 失敗時，Bot 不會假裝操作成功。

## 後續版本

- 重複提醒。
- 「完成了嗎？」確認流程。
- 「晚一小時」或「明天再提醒」等延後功能。
- 共享待辦或家庭群組功能。
- 語音與圖片理解。
- 本機 Ollama 模型與 Cloudflare Tunnel 整合。
- 長期對話搜尋。
